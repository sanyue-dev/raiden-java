# Issue Tracker: GitHub

Issues and PRDs for this repo live in GitHub Issues for `sanyue-dev/raiden-java`.

## Conventions

- **Create an issue**: use GitHub Issues in `sanyue-dev/raiden-java`.
- **Read an issue**: fetch the issue body, labels, reporter, dates, and comments before recommending a triage outcome.
- **List issues**: inspect open issues by label and by unlabeled state when looking for work that needs attention.
- **Comment on an issue**: every triage comment must start with:

  ```markdown
  > *This was generated by AI during triage.*
  ```

- **Apply or remove labels**: use the label strings defined in `docs/agents/triage-labels.md`.
- **Close an issue**: only after the chosen triage state requires closure, such as `wontfix`.

## When a skill says "publish to the issue tracker"

Create a GitHub issue in `sanyue-dev/raiden-java`.

## When a skill says "fetch the relevant ticket"

Fetch the matching GitHub issue from `sanyue-dev/raiden-java`, including comments and labels.
